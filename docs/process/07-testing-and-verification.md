# 07 测试与验证

## 1. 测试目标

验证“无表临时用电终止”业务模块在课程设计范围内能够完成核心业务闭环：

受理 -> 派工 -> 勘查 -> 终止合同 -> 停电 -> 费用确定 -> 结清 -> 信息归档 -> 回访 -> 最终归档。

## 2. 测试原则

1. 聚焦课程设计目标，重点验证业务准确性和系统可运行性。
2. 不引入生产级性能、安全、容灾、外部联调测试。
3. 自动化测试覆盖关键路径，避免仅依赖手工点击验证。
4. 测试数据以演示和验收为目的，保持可读、可复现。

## 3. 自动化测试结构

| 测试类 | 类型 | 验证内容 |
| --- | --- | --- |
| `TerminationApplicationServiceTest` | 服务层测试 | 费用规则、完整流程流转、非法状态控制 |
| `TerminationApiControllerTest` | 接口层测试 | REST 查询、创建、流程操作、异常响应 |
| `TerminationPageControllerTest` | 页面层测试 | 台账页渲染、详情页渲染、新增受理表单跳转 |
| `PowerSupplyTerminationApplicationSmokeTest` | 启动冒烟测试 | Spring Boot 上下文启动、样例页面访问 |

## 4. 覆盖范围

| 层级 | 已覆盖内容 |
| --- | --- |

| 异常控制 | 非法状态操作抛出业务异常或返回错误响应 |
