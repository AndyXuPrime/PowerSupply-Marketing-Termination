# 03 实施计划

## 阶段 1：工程与文档骨架

- 建立 Maven Spring Boot 项目。
- 建立 `docs/process/` 过程性文档。
- 补充 README、运行方式和业务范围。

## 阶段 2：后端业务实现

- 建立申请实体、流程状态、仓储层。
- 实现受理、派工、勘查、合同终止、停电、费用确定、费用结清、归档和回访服务。
- 提供 REST API，便于前后端分离扩展。

## 阶段 3：前端页面实现

- 申请列表与检索。
- 新增业务受理表单。
- 申请详情、流程时间线、环节操作面板。
- 费用、资料、回访和归档信息展示。

## 阶段 4：验证与同步

- 运行 Maven 测试。
- 启动本地服务验证页面。
- 分阶段提交 Git，并同步到 GitHub 远端仓库。
