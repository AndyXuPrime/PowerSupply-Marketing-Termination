## 3. 重点问题说明

### 3.1 Maven 本地仓库权限问题

项目所在环境的全局 Maven 配置将本地仓库固定到系统目录，该目录当前无写权限。为避免修改用户全局环境，项目采用本仓库内 `.mvn/settings.xml` 固定本地仓库路径。后续构建统一使用：

```powershell
mvn -s .mvn/settings.xml test
```

该方案对项目内构建生效，不影响机器上的其他 Maven 项目。

### 3.2 Thymeleaf 详情页模型冲突

`application` 在 Thymeleaf/Spring Web 环境中容易与内置 application 作用域发生语义冲突。详情页原先使用 `${application.customerName}` 等表达式时，自动化测试显示业务对象字段未被正确渲染。

处理方式为将控制器模型属性更名为 `terminationApplication`，并同步调整页面表达式。该命名更贴合业务含义，也降低了与框架内置对象冲突的风险。

### 3.3 测试范围控制

本项目为课程设计，不按生产系统要求建设测试体系。测试重点放在以下内容：

- 核心费用规则是否正确。
- 流程是否按规定顺序推进。
- 非法流程操作是否被拦截。
- 页面是否可访问并显示业务数据。
- Spring Boot 应用是否可正常启动。

不纳入本阶段范围的内容包括性能压测、复杂权限测试、安全渗透测试、外部系统联调测试。

上述问题均已在当前阶段完成处理。最新验证命令 `mvn -s .mvn/settings.xml test` 已通过，系统具备课程设计演示和验收所需的基本稳定性。
