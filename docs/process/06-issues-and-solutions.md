# 06 问题与解决方案

## 1. 需求文档读取困难

- 问题：`PDF` 是 WPS 生成文件，正文不是直接可见文本，常规字节搜索和直接解析都不稳定。
- 影响：无法快速定位“无表临时用电终止”章节。
- 解决方案：改读同目录下的 `.doc` 版本，直接抽取 UTF-16 文本并定位到 `BM01_009/无表临时用电终止` 章节。
- 结果：成功提取流程、规则、费用约束和归档要求。

## 2. 本地缺少 PDF 解析依赖

- 问题：环境里没有 `pypdf`、`pdfplumber` 等库，联网安装也受限。
- 影响：不能依赖现成 Python 库完成文档解析。
- 解决方案：不再强行装库，转而使用本地无依赖方式处理 `.doc` 文本。
- 结果：避免了额外依赖和安装风险。

## 3. Maven 默认仓库权限不足

- 问题：全局 `settings.xml` 把本地仓库写死到 `D:\MyMaven\apache-maven-3.9.9\repo`，当前环境没有写权限。
- 影响：`mvn test` 和 `mvn package` 直接失败。
- 解决方案：在项目内新增 `.mvn/settings.xml`，把本地仓库切换到项目目录下的 `.m2/repository`。
- 结果：构建不再依赖系统级 Maven 仓库路径。

## 4. 首次构建需要联网下载依赖和插件

- 问题：Spring Boot 父 POM、Maven 插件和相关依赖第一次都不在本地缓存里。
- 影响：离线状态下无法完成构建。
- 解决方案：申请一次联网构建，把依赖和插件下载到项目本地仓库中。
- 结果：`mvn test` 和 `mvn -DskipTests package` 成功通过。

## 5. 样例数据初始化触发校验失败

- 问题：`DataInitializer` 直接向仓库写入实体时，`applicationNo` 还没生成，实体校验先触发了。
- 影响：应用启动失败。
- 解决方案：取消对系统自动生成字段的强校验，并让初始化逻辑走 `service.create(...)`，由业务层统一生成申请编号和客户编号。
- 结果：样例数据正常写入，应用可启动。

## 6. Thymeleaf 模板出现弃用写法提醒

- 问题：页面里用了旧式 fragment 语法。
- 影响：启动时出现弃用警告。
- 解决方案：改为 `~{fragments/nav :: nav('list')}` 的标准写法。
- 结果：消除了模板弃用提示。

## 7. 后台启动验证不稳定

- 问题：直接用隐藏窗口方式拉起 Java 进程时，前台探测不稳定。
- 影响：看起来像是服务未启动。
- 解决方案：先通过前台日志确认应用能正常启动，再用 PowerShell job 验证 `http://localhost:8080/applications` 返回 `200`。
- 结果：确认前后端链路已跑通。

## 8. Windows 终端中文输出乱码

- 问题：PowerShell 直接 `Get-Content` 某些文件时，中文显示混乱。
- 影响：阅读源码和模板不方便。
- 解决方案：使用 Python 按 UTF-8 重新读取并必要时重写文件。
- 结果：中文内容恢复正常，后续编辑稳定。
