# 无表临时用电终止业务模块使用说明

## 1. 文档信息

| 项目 | 内容 |
| --- | --- |
| 文档名称 | 无表临时用电终止业务模块使用说明 |
| 项目名称 | PowerSupply-Marketing-Termination |
| 适用对象 | 课程设计演示人员、测试人员、评审教师 |
| 适用范围 | `BM01_009/无表临时用电终止` 业务模块 |
| 系统类型 | Spring Boot 单体 Web 应用 |

## 2. 系统概述

本系统用于模拟供电企业营销业务中的“无表临时用电终止”流程。该业务面向已经办理无表临时用电的客户，在客户申请终止、临时用电期限届满、或不再满足继续供电条件时，由供电企业办理现场核实、合同终止、停电、费用确认、费用结清、资料归档和客户回访等操作。

本项目为课程设计系统，目标是完整表达业务主流程和关键规则，不承担真实电力营销系统的生产办理职责。

## 3. 启动与访问

### 3.1 启动命令

在项目根目录执行：

```powershell
mvn -s .mvn/settings.xml spring-boot:run
```

如果已经完成打包，也可以执行：

```powershell
java -jar target/power-supply-marketing-termination-0.1.0-SNAPSHOT.jar
```

### 3.2 页面地址

| 功能 | 地址 |
| --- | --- |
| 业务前端 | `http://localhost:8080/applications` |
| 系统默认入口 | `http://localhost:8080` |
| H2 数据库控制台 | `http://localhost:8080/h2-console` |

### 3.3 H2 控制台参数

| 参数 | 填写值 |
| --- | --- |
| JDBC URL | `jdbc:h2:file:./data/termination-db` |

## 5. 完整操作流程

### 5.1 新增业务受理

1. 访问 `http://localhost:8080/applications`。
2. 点击页面右上方“新增受理”，或下拉到“新增受理”表单。
3. 填写客户和临时用电信息。
4. 点击“保存受理”。
5. 系统创建申请并跳转到申请详情页。

必填或关键字段说明：

| 字段 | 含义 | 示例 |
| --- | --- | --- |
| 客户名称 | 办理临时用电终止的客户或项目主体名称 | 城南临时施工项目部 |
| 联系人 | 客户侧业务联系人 | 张三 |
| 联系电话 | 联系人手机号或固定电话 | 13800000000 |
| 客户地址 | 临时用电现场地址 | 某区施工现场 |
| 行政区 | 客户所属供电服务区域 | 城南区 |
| 供电单位 | 办理该业务的供电所或供电单位 | 城南供电所 |
| 项目名称 | 临时用电对应的工程或项目名称 | 市政道路改造临电项目 |
| 项目类型 | 临时用电项目类别 | 基建工地、市政建设、农田水利 |
| 用电性质 | 表示该用电业务类型 | 无表临时用电 |
| 合同号 | 临时供用电合同编号 | HT-2026-001 |
| 证件号码 | 客户身份或单位相关证件编号 | 统一社会信用代码或身份证号 |
| 申请来源 | 业务申请渠道 | 营业厅受理、电话申请 |
| 受理区域 | 办理业务的区域 | 城南营业厅 |
| 临时容量 | 临时用电合同容量，单位为 kW | 120 |
| 预收电费 | 办理无表临时用电时预收的电费金额，单位为元 | 3600 |
| 约定期限 | 临时用电合同约定使用天数 | 30 |
| 申请终止时间 | 客户提出终止用电的日期 | 2026-05-13 |
| 备注 | 风险提示、欠费提示或其他说明 | 客户申请提前终止 |

系统自动生成：

| 字段 | 生成规则 |
| --- | --- |
| 申请编号 | `BM01_009-` 加当前时间 |
| 客户编号 | `C` 加当前时间 |
| 初始状态 | `业务受理` |
| 受理日期 | 当前系统日期 |

### 5.2 勘查派工

办理条件：申请状态为“业务受理”。

操作步骤：

1. 在详情页找到“勘查派工”操作卡片。
2. 填写勘查人员姓名。
3. 点击“派工”。

系统处理结果：

| 项目 | 结果 |
| --- | --- |
| 当前状态 | 变更为“勘查派工” |

| 备注 | 追加“已派工至勘查环节” |

### 5.3 现场勘查

办理条件：申请状态为“勘查派工”。

操作步骤：

1. 在详情页填写“勘查结论”。
2. 如现场地址与受理地址不同，可填写“客户地址修正”。
3. 点击“保存勘查”。

系统处理结果：

| 项目 | 结果 |
| --- | --- |
| 当前状态 | 变更为“现场勘查” |
| 勘查日期 | 自动记录当前日期 |
| 勘查结果 | 保存现场核实说明 |
| 客户地址 | 如填写修正地址，则覆盖原地址 |
| 资料状态 | 更新为“已完成现场勘查，等待合同终止” |

### 5.4 终止合同

办理条件：申请状态为“终止合同”。

### 5.6 确定费用

办理条件：申请状态为“停电”。

操作步骤：

1. 填写“实际使用天数”。
2. 点击“计算费用”。
3. 系统根据约定期限和预收电费自动计算退费或补收金额。

费用规则：

| 场景 | 判断方式 | 结果 |
| --- | --- | --- |
| 提前终止且未达半期 | 实际使用天数 < 约定天数 / 2 | 退还预收电费的一半 |
| 超过半期但未超期 | 约定天数 / 2 <= 实际使用天数 <= 约定天数 | 不退不补 |
| 超过约定期限 | 实际使用天数 > 约定天数 | 按超期天数补收 |

示例：

| 预收电费 | 约定天数 | 实际天数 | 计算结果 |
| --- | --- | --- | --- |
| 3600 元 | 30 天 | 14 天 | 退费 1800 元 |
| 3600 元 | 30 天 | 20 天 | 不退不补 |
| 3600 元 | 30 天 | 35 天 | 日均 120 元，超期 5 天，补收 600 元 |

系统处理结果：

| 项目 | 结果 |
| --- | --- |
| 当前状态 | 变更为“确定费用” |
| 实际使用天数 | 保存输入值，保留两位小数 |
| 退费金额 | 根据规则自动计算 |
| 应收金额 | 根据规则自动计算 |
| 终止费用 | 当前系统中与补收金额一致 |
| 费用说明 | 自动生成计算说明 |

## 7. 专业内容含义说明

### 7.1 业务术语

| 术语 | 含义 |
| --- | --- |
| 无表临时用电 | 不具备安装计量表计条件、按容量和期限估算电费的临时用电方式 |
| 临时用电 | 为基建工地、市政建设、农田水利等非永久性项目提供的临时供电 |
| 用电终止 | 客户或供电企业结束该临时用电关系，停止供电并完成后续费用和资料处理 |
| 临时供用电合同 | 供电企业与客户之间关于临时用电容量、期限、费用等约定的合同 |
| 预收电费 | 办理无表临时用电时预先收取的电费 |
| 实际使用天数 | 客户实际使用临时电源的天数，是退补费计算的核心依据 |
| 约定期限 | 临时供用电合同中约定的用电天数 |
| 超期天数 | 实际使用天数超过约定期限的部分 |
| 退费金额 | 因提前终止且满足退费条件而应退还客户的金额 |
| 应收金额 | 因超期使用或其他原因需要向客户补收的金额 |
| 结清费用 | 完成退费、补收、欠费确认和凭证记录 |
| 信息归档 | 将业务资料、档案位置和资料状态登记到系统 |
| 客户回访 | 业务办理后对客户进行结果确认或满意度回访 |
| 最终归档 | 业务资料完整、费用处理完成、客户回访完成后的最终收尾 |

## 9. 常见问题

### 9.1 为什么只能看到一个操作卡片

详情页只显示当前状态允许执行的下一步操作。例如状态为“业务受理”时，只能执行“勘查派工”，不能直接执行“停电”或“归档”。

### 9.2 为什么费用计算需要实际使用天数

无表临时用电没有表计电量，本课程设计按“预收电费、约定期限、实际使用天数”进行退补费判断，因此实际使用天数是费用确定环节的必要输入。

### 9.3 为什么没有登录和权限控制

本项目为课程设计，重点在业务流程建模和系统实现，不建设生产级统一认证和权限矩阵。页面中的角色通过业务环节说明体现。

### 9.4 为什么没有外部系统联动

真实业务可能涉及营销系统、计费系统、档案系统、短信平台等。本项目只实现本业务模块的核心闭环，不模拟外部系统调用。

### 9.5 如何清空演示数据

停止应用后，删除 `data/` 目录下的 H2 数据文件，再重新启动系统。重新启动后系统会按初始化逻辑生成演示数据。
